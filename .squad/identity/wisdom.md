# Team Wisdom

Reusable patterns and heuristics learned through work. NOT transcripts — each entry is a distilled, actionable insight.

## Patterns

<!-- Append entries below. Format: **Pattern:** description. **Context:** when it applies. -->
**Pattern:** When `/session/animation` only carries semantic animation ids, audit the generated runtime sidecar before blaming backend transport for missing motion. **Context:** The backend can be correct while the actual drop lives in frontend binding from locally resolved runtime `channels` onto VRM bones.
**Pattern:** Treat wrist and finger delivery as separate seams during humanoid playback audits. **Context:** Lower-arm hints and wrist channels can already be present and active even when finger channels remain exported-but-unbound, so hand fidelity checks should distinguish wrist mapping from finger-bone wiring.
**Pattern:** If generated humanoid sidecars already preserve broader body-region channels, keep the backend semantic-only and audit frontend runtime binding coverage before widening transport or payload contracts. **Context:** Missing arm or hand motion can come from a selective binding table in `humanoidChannelPlayback.ts` even when export, sidecar hydration, and semantic id delivery are already correct.
**Pattern:** Bind only directly mappable finger `*.stretched` channels first and leave spread, transform hints, twist, lower-body, jaw, eye, and root channels unbound until their scope is explicitly approved. **Context:** Conservative finger stretch bindings improve visible hand articulation without disturbing already-correct torso, upper-arm, elbow, wrist, and lower-arm playback.
**Pattern:** When spread samples are finite and non-trivial across `idle.default`, `listen.loop`, and `speak.loop`, the conservative first spread mapping is one root binding per digit: thumb spread on the thumb metacarpal and index, middle, ring, and little spread on the proximal finger bone with mirrored handedness signs. **Context:** This keeps the hand slice falsifiable and narrow, preserves the existing arm and finger stretch behavior, and gives live `/display/` verification a clear success target of 10 active spread bindings.
**Pattern:** If both regenerated `idle.default` and a local reference clip such as `gesture.punch.once` resolve in the frontend catalog, treat welded shoulder behavior as a runtime weighting problem before reopening export or backend transport. **Context:** Matching reference availability plus local payload resolution is enough to shift the next check toward `humanoidChannelPlayback.ts` influence tuning rather than another export or contract audit.
**Pattern:** A spaced source filename can truncate exported raw-source provenance text without invalidating the generated clip for validation. **Context:** Keep the clip usable for arm and hand reference checks if the DSL and runtime artifacts are present and resolvable, then fix provenance formatting separately if exact source labeling becomes important.
**Pattern:** When a shared reference clip already resolves locally but the next question is whether wrist and lower-arm delivery really engage during live playback, add a dev-only frontend override panel on the display surface and use direct browser A/B playback as the discriminating check. **Context:** This keeps backend animation contracts unchanged while proving whether `gesture.punch.once` actually activates wrist channels and lower-arm quaternion bindings in the current runtime.
**Pattern:** If forced punch playback activates wrist channels and sampled lower-arm quaternion bindings but does not reveal one singular new runtime defect, log the pass as validation rather than another arm-chain fix. **Context:** The right follow-up then is targeted shoulder, elbow, or wrist fidelity tuning, not a speculative patch justified only by already-known weighting concerns.
**Pattern:** If a high-motion generated clip such as `gesture.punch.once` already carries non-trivial limb ranges in the checked-in runtime artifact, treat a global exporter-drop hypothesis as falsified before reopening exporter design. **Context:** A clip-binding exporter can still preserve real forearm, hand, leg, foot, elbow-flex, and lower-arm quaternion signal, so missing visible motion may live in runtime binding coverage or weighting instead.
**Pattern:** Do not use `idle.default` alone to judge missing arm-bend delivery when its forearm and elbow channels are present but nearly flat. **Context:** Pair idle with a stronger reference clip like `gesture.punch.once` so weak authored signal is not misread as a broken exporter or transport seam.
**Pattern:** When Jason supplies a concrete Unity endpoint screenshot and the lower-body plus hand baseline is already stable, keep the correction local to `gesture.punch.once` runtime weighting and use the screenshot as the live validation target. **Context:** This lets shoulder carry, wrist alignment, and punch-specific upper-arm carry be tuned without reopening exporter work or regressing the landed leg and finger behavior.
**Pattern:** If a second punch pass still finishes with a low guard while lower-arm quaternion hints remain active, treat the remaining mismatch as clip-specific upper-arm `down_up` plus `front_back` carry before blaming quaternion interpretation. **Context:** Elbows and forearms can still need more forward and slightly higher travel even when lower-arm quaternion delivery is already working end to end.
**Pattern:** When repeated runtime-only axis or carry tweaks stop producing one honest fix across both idle and punch, checkpoint an additive exporter experiment instead of another remap guess: preserve the current muscle-space path, sample `Animator.GetBoneTransform(HumanBodyBones)` side by side for one high-signal clip first, compare the final frame on the dev-only browser override surface, and keep backend contracts semantic-only. **Context:** This turns a fuzzy arm-space problem into an evidence-gathering export comparison while keeping the machine-transition handoff and commit scope clean.
**Pattern:** Treat `.tmp-unity-temp` as cache noise during Unity export investigations and keep it out of commits, especially when checkpointing work for a machine transition. **Context:** Exporter experiments can require repo-local temp redirection, but that should not leak temp-project artifacts into squad continuity or git history.
